# 🚧 工程劳动力计划生成器

一个现代化的网页版工程劳动力计划生成工具，基于React + TypeScript + Tailwind CSS开发。

## ✨ 功能特点

### 🎯 核心功能
- **多工程类型支持**：路基工程、桥梁工程、路面工程、隧道工程、房建工程
- **智能生成算法**：根据工程进度自动生成各工种劳动力需求曲线
- **手动配置模式**：支持自定义各工种人数配置
- **多项目汇总**：自动汇总不同工程类型的劳动力需求
- **数据导出**：生成CSV格式报表，可直接在Excel中打开

### 🎨 界面特色
- **现代化设计**：采用天气应用风格的配色方案
- **响应式布局**：适配不同屏幕尺寸
- **直观操作**：卡片式项目选择，标签页配置界面
- **实时反馈**：操作状态即时显示

## 📖 使用说明

### 1️⃣ 选择工程类型
- 在左侧选择要规划的工程类型（可多选）
- 每个工程类型包含不同的施工阶段

### 2️⃣ 配置工程参数
- **时间范围**：设置各阶段的开始和结束时间
- **队伍数量**：配置施工队伍数量（影响总人数）
- **配置模式**：
  - 🤖 智能生成：系统自动根据工程进度生成劳动力曲线
  - 📝 手动配置：自定义各工种的具体人数

### 3️⃣ 生成计划
- 点击"🚀 生成计划"按钮
- 系统会自动计算并导出CSV格式的劳动力计划表

### 4️⃣ 快速测试
- 点击"📊 快速测试"可生成示例数据，用于功能测试

- 普通工、司机、支护工

## 🧮 智能生成算法

系统采用与原版Python应用完全一致的算法：

- **模板工**：前期和中期需求较高
- **混凝土工**：中期需求最高
- **钢筋工**：前期和中期需求较高
- **支架工**：前期和中期需求较高
- **测量工**：前期和后期需求较高
- **电焊工**：中期需求较高
- **泥瓦工**：后期需求较高
- **电工**：均匀分布，略中后期增加
- **普工**：全程都需要，中期需求最高

## 🔧 开发说明

### 添加新的工程类型
1. 在 `constants/workforce.ts` 中添加新的工程类型配置
2. 在类型定义中添加对应的工种类型
3. 更新智能生成算法（如需要）

### 修改配色方案
编辑 `constants/workforce.ts` 中的 `MODERN_COLORS` 常量

### 调整生成算法
修改 `utils/workforceGenerator.ts` 中的相关函数

## 📝 注意事项

1. 导出的文件为CSV格式，可直接用Excel打开
2. 时间范围设置要合理，结束时间必须晚于开始时间
3. 建议先使用"快速测试"功能验证系统正常工作
4. 大型项目建议分阶段生成计划，便于管理
